# CLAUDE.md

## Project overview

agent-harness is a CLI tool that automates the factory playbook's task-to-intent pipeline. It connects to a conflict-free orchestrator, polls for ready tasks, sends them to Claude API to generate AST-CRDT intents, and applies those intents back to the orchestrator.

## Origin

Generated by the playbook-factory pipeline (run 9, 2026-04-04) in 3 minutes:
- 31 HTTP intents applied to a conflict-free orchestrator in sharded mode
- 0 intent failures, 0 manual fixups
- Source synced via `ast-crdt project sync --from-server`
- First CLI tool (not a web service) built by the factory
- First factory product that is itself part of the factory pipeline

## Architecture

- `types.rs` -- Shared types: AgentConfig, Task, IntentWrapper, Claude API request/response structs
- `orchestrator_client.rs` -- HTTP client wrapping the conflict-free `/a2a/*` endpoints (tasks, source, intents)
- `claude_client.rs` -- HTTP client for the Anthropic Messages API; builds prompts from source + task description
- `runner.rs` -- Main loop: poll ready tasks, claim, generate intents via Claude, apply, mark complete
- `main.rs` -- CLI parsing via clap, config assembly, entry point

## Key patterns

- Uses `reqwest` for all HTTP (orchestrator + Claude API)
- `ANTHROPIC_API_KEY` from environment, not CLI args (security)
- Orchestrator auth token passed as CLI arg (matches orchestrator's `--auth-token`)
- Runner loop exits when no ready tasks remain (batch mode, not daemon)
- Failed intent parsing skips the task (logs error, marks complete to avoid blocking the DAG)

## Build and run

```bash
cargo build --release
export ANTHROPIC_API_KEY=sk-...
./target/release/agent-harness http://localhost:8080 TOKEN MODEL AGENT_NAME
```

## Intent format

The Claude prompt asks for intents in the conflict-free HTTP API format:
```json
[{"intent": {"type": "add_function", "name": "...", ...}}, ...]
```

See the factory playbook's intent reference and the orchestrator's `/openapi.json` for the full schema.
