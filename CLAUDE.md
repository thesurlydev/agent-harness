# CLAUDE.md

## Project overview

agent-harness is a CLI tool that automates the factory playbook's task-to-intent pipeline. It connects to a conflict-free orchestrator, polls for ready tasks, sends them to Claude API to generate AST-CRDT intents, and applies those intents back to the orchestrator.

## Origin

Generated by the playbook-factory pipeline (run 9, 2026-04-04) in 3 minutes:
- 31 HTTP intents applied to a conflict-free orchestrator in sharded mode
- 0 intent failures, 0 manual fixups
- Source synced via `ast-crdt project sync --from-server`
- First CLI tool (not a web service) built by the factory
- First factory product that is itself part of the factory pipeline

## Architecture

- `types.rs` -- Shared types: AgentConfig, AgentMode, Task, IntentWrapper, PrdIntentWrapper, PrdAgentRole, Claude API types, PRD response types (completeness, verification, artifacts)
- `orchestrator_client.rs` -- HTTP client wrapping the conflict-free `/a2a/*` endpoints (tasks, source, intents, PRD source/intents/completeness/verify/artifacts)
- `claude_client.rs` -- HTTP client for the Anthropic Messages API; builds prompts for code intents, fixes, and PRD intents
- `runner.rs` -- Code mode main loop: poll ready tasks, claim, generate intents via Claude, apply, mark complete; includes compile-fix loop
- `prd_runner.rs` -- PRD mode: 3-round enrichment protocol (parallel enrichment, adversarial review, contract lock) with completeness checking, verification, and artifact derivation
- `main.rs` -- CLI parsing via clap, config assembly, mode dispatch

## Key patterns

- Uses `reqwest` for all HTTP (orchestrator + Claude API)
- `ANTHROPIC_API_KEY` from environment, not CLI args (security)
- Orchestrator auth token passed as CLI arg (matches orchestrator's `--auth-token`)
- Two modes: `--mode code` (default) polls tasks and generates AST-CRDT intents; `--mode prd` runs the 3-round PRD enrichment pipeline
- Code runner loop exits when no ready tasks remain (batch mode, not daemon)
- Failed intent parsing skips the task (logs error, marks complete to avoid blocking the DAG)
- PRD enrichment dispatches agents in parallel via `tokio::spawn`, with role-specific prompts from `prompts/prd/`
- Compile-fix loop (enabled by `--project-dir`) syncs source to disk and iterates `cargo check` + Claude fix calls up to `--max-fix-rounds` times

## Build and run

```bash
cargo build --release
export ANTHROPIC_API_KEY=sk-...

# Code mode (default): generate AST-CRDT intents from tasks
./target/release/agent-harness http://localhost:8080 TOKEN MODEL AGENT_NAME

# Code mode with compile-fix loop
./target/release/agent-harness http://localhost:8080 TOKEN MODEL AGENT_NAME \
  --project-dir ./my-project --max-fix-rounds 5

# PRD enrichment mode
./target/release/agent-harness http://localhost:8080 TOKEN MODEL AGENT_NAME \
  --mode prd --app-type full-stack --project-dir ./my-project
```

## Intent format

The Claude prompt asks for intents in the conflict-free HTTP API format:
```json
[{"intent": {"type": "add_function", "name": "...", ...}}, ...]
```

See the factory playbook's intent reference and the orchestrator's `/openapi.json` for the full schema.
